## Installation

```package-install
npm i @orpc/contract
```

## Define the Contract

The API almost the same as the `server` except that contract will not define the handler.

```ts
import { oc } from '@orpc/contract'

export const appRouterContract = oc.router({
    ping: oc
        .route({ method: 'GET', path: '/ping' }) // optional
        .input(z.object({ value: z.string() }))   // optional
        .output(z.string()),                      // optional

    user: oc.prefix('/users').router({
        find: oc 
            .input(z.object({ id: z.string() }))
            .output(z.object({ id: z.string(), name: z.string() })),
    })
})
```

## Implement the Contract

All `server` features are available, except the input, output, and route parts, which are defined in the contract.

```ts
import { os } from '@orpc/server'
import { appRouterContract } from './contract'

const osw = os.contract(appRouterContract)

export const appRouter = osw.router({
    ping: osw.ping.handler(({ input }) => input.value),
    user: {
        find: osw.user.find.handler(({ input }) => {
            return {
                id: input.id,
                name: 'dinwwwh',
            }
        }),
    }
})
```
