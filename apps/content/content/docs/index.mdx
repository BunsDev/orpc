## Introduction
ORPC is splitting into three parts: `server`, `client`, and `contract` (optional). 
Where server the the place you define app logics, and client communicate with server to call procedures. 
Whe first class support contract first design by using `contract` (This is optional).

## Installations
For the first time, we only cover the `server` and `client` parts.

```package-install
npm i @orpc/server @orpc/client
```

## Define your first router

```ts
import { initORPC } from '@orpc/server'
import { z } from 'zod'

export type Context = {}

const orpc = initORPC.context<Context>()

const router = orpc.router({
  ping: orpc
    .route({method: 'GET', 'path': '/ping'}) // optional
    .input(z.object({ name: z.string() })) // optional
    .handler(async (input, context, meta) => `pong ${input.name}`),
})
```

## Start your server

Here I use a simple package `@hono/node-server` to start a node server with Fetch API.

```ts
import { serve } from '@hono/node-server'
import { createRouterHandler } from '@orpc/server'
import { fetchHandler } from '@orpc/server/fetch'

// For performance purpose, we should create a global router handler
const handler = createRouterHandler({
  router,
  serverless: false, // Set it to true will improve cold start times
})

serve({
  fetch(request){
    fetchHandler({
      request,
      handler,
      prefix: '/',
      context: {},
    })
  },
  port: 2026
})
```

Now start the server and visit http://localhost:2026/ping?name=dinwwwh to se the result

## Client usage

Fully typed client is also provided, you can use it in any environment.

```ts
import { createORPCClient } from '@orpc/client'

const client = createORPCClient<typeof router>({
  baseURL: 'http://localhost:2026',
})

client.ping({ name: 'dinwwwh' }).then((output) => console.log(output))
```

That's all very convenient.
